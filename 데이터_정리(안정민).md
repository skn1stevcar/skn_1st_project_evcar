## 1. 전국 전기차 충전소 표준데이터

- **제공기관**: 한국환경공단
- **포맷**: XML (OpenAPI 제공)
- **End Point**: `https://apis.data.go.kr/B552584/EvCharger`
- **링크**: <https://www.data.go.kr/data/15013115/standard.do>
- **설명**: 지자체·민간이 운영하는 전국 전기차 충전소의 위치·상태·운영정보를 실시간 제공. EV 사용자의 충전 편의 향상 목적.

### 제공 기능 (상세기능 2종, 각 일일 1,000건)

| NO | 상세기능 | 설명 |
|----|----------|------|
| 1 | `/getChargerStatus` | 충전기 상태 실시간 제공 (충전소ID, 충전기ID, 상태정보) |
| 2 | `/getChargerInfo` | 충전소 정보 제공 (충전소명, 위치, 상태, 운영시간, 운영기관명, 충전기 용량) |

### 요청 파라미터

| 항목명 | 설명 |
|--------|------|
| `ServiceKey` | 인증키 (서비스키) |
| `pageNo` | 페이지 번호 |
| `numOfRows` | 한 페이지 결과 수 |
| `period` | 상태갱신 기간 |
| `zcode` | 지역구분 코드 |

### 응답 항목

| 항목명 | 설명 |
|--------|------|
| `resultCode` | 결과코드 |
| `resultMsg` | 결과메시지 |
| `pageNo` | 페이지 번호 |
| `totalCount` | 데이터 총 개수 |
| `busiId` | 기관 아이디 |
| `statId` | 충전소 ID |
| `chgerId` | 충전기 ID |
| `stat` | 충전기 상태 |
| `statUpdDt` | 상태갱신 일시 |

> ℹ️ `getChargerInfo`는 위 항목 외에 충전소명, 주소, 위도/경도, 충전기 타입, 운영시간, 운영기관명, 충전기 용량 등을 추가로 제공합니다.

## 🔗 세 데이터를 엮는 방법 (프로젝트 활용 가이드)

> **비전공자용 한 줄 요약**
> 서로 다른 표(데이터)를 합치려면 **"공통으로 들어맞는 열(column)"** 이 있어야 합니다.
> 엑셀에서 VLOOKUP으로 두 시트를 합칠 때 "사번"이나 "지역명"으로 짝을 맞추는 것과 똑같습니다.
> 우리 데이터는 **① 지역(어디)** 과 **② 날짜·시간(언제)** 이라는 두 개의 공통 열로 연결할 수 있습니다.

### 1) 무엇을 기준으로 엮을 수 있나? (연결 열)

| 연결 기준 | 전기차 대수 | 전국 교통량 | 충전소 데이터 | 비전공자 설명 |
|-----------|------------|------------|--------------|----------------|
| **지역 (어디)** | `서울`~`제주` (시·도 열 이름) | 영업소/노선 코드 → 시·도로 변환 | 주소·`zcode`(지역코드)·위경도 → 시·도로 변환 | "이 데이터가 **어느 동네** 것인지"를 맞춰주는 열. 셋 다 시·도(17개 광역) 단위로 통일하면 한 표로 합쳐집니다. |
| **시간 (언제)** | `기준일` (월말) | `집계일자`+`집계시` | `statUpdDt`(상태갱신일시) | "이게 **언제** 기록된 값인지"를 맞춰주는 열. 가장 거친 단위인 **'월'** 로 통일하면 비교가 쉽습니다. |

### 2) 각 데이터에서 실제로 쓸 핵심 컬럼

| 데이터 | 꼭 쓸 컬럼 | 무슨 값으로 가공할까 |
|--------|-----------|----------------------|
| **전기차 대수** | `기준일`, 시·도 17개 열 | 지역별 전기차 보급 대수 (수요 지표) |
| **전국 교통량** | `집계일자`, `집계시`, `고속도로운영기관구분코드`(지역 추정), `총교통량`, `1~6종교통량` | 지역·시간대별 통행량 (도로 이용량 지표) |
| **충전소 데이터** | `statId`(충전소 수 세기), `zcode`/주소(지역), 위경도, `stat`(가동상태) | 지역별 충전소·충전기 개수, 가동률 (공급 지표) |

### 3) 합치는 순서 (가장 쉬운 방법)

1. **공통 단위 정하기** — 지역은 **시·도(17개)**, 시간은 **월** 로 통일합니다. (가장 거친 단위에 맞추면 빈칸이 안 생깁니다.)
2. **각 데이터를 시·도 × 월 단위로 요약(집계)** 합니다.
   - 교통량: 영업소/좌표를 시·도로 바꾼 뒤, 한 달치를 더해 "월 총교통량"으로 만듭니다.
   - 충전소: 주소·좌표·`zcode`로 시·도를 정하고, 충전소(`statId`) 개수를 셉니다.
   - 전기차: 이미 시·도 × 월말 형태라 그대로 사용합니다.
3. **시·도(+월)를 열쇠 삼아 옆으로 이어 붙입니다.** → 한 행에 `지역 | 월 | 전기차대수 | 교통량 | 충전소수` 가 모입니다.

### 4) 합치고 나면 무엇을 볼 수 있나 (분석 아이디어)

| 질문 | 사용 컬럼 조합 | 의미 |
|------|----------------|------|
| 전기차가 많은 지역에 충전소도 충분한가? | 전기차대수 ÷ 충전소수 | **충전소 1곳당 전기차 대수** — 값이 클수록 충전 인프라 부족 의심 |
| 교통량 많은 지역이 충전소도 많은가? | 교통량 vs 충전소수 | 통행량 대비 충전 인프라 적정성 |
| 전기차가 빠르게 느는 지역은? | 월별 전기차대수 증가폭 | 향후 충전소를 늘려야 할 우선 지역 |

> ⚠️ **현실적 주의**
> - **기간이 다릅니다**: 전기차(2024.08~2025.02)와 교통량(2026.05)은 겹치는 달이 없어, 같은 달끼리 비교하려면 **동일 기간 데이터를 추가로 받아야** 합니다.
> - **지역 변환이 필요합니다**: 교통량(영업소/노선)과 충전소(좌표·주소)는 시·도가 직접 안 적혀 있어, **"좌표/주소 → 시·도" 변환 작업**이 한 번 필요합니다.
